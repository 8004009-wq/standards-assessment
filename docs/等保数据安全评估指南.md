# 等保数据安全评估指南

## 📋 标准概述

**标准名称**: 网络安全等级保护数据安全基本要求  
**标准编号**: GB/T 22239-2019（数据安全部分）  
**适用范围**: 各等级信息系统的数据安全保护  

本评估模板基于网络安全等级保护 2.0 的数据安全要求设计，覆盖数据全生命周期的安全保护要求。

## ✅ 评估方法

### 1. 评估准备

- [ ] 确定评估对象（信息系统/业务应用）
- [ ] 确定等保级别（二级/三级）
- [ ] 组建评估团队
- [ ] 准备评估工具和文档

### 2. 评估执行

对每个评估项进行判断：

| 评级 | 说明 | 得分比例 |
|------|------|---------|
| **符合** | 完全满足要求，有完整证据 | 100% |
| **部分符合** | 部分满足要求，有改进空间 | 50% |
| **不符合** | 不满足要求或缺失 | 0% |
| **不适用** | 该项不适用于当前系统 | 不计分 |

### 3. 证据收集

每个评估项应收集以下证据：

- **制度文档**: 管理制度、操作规程、应急预案等
- **技术配置**: 系统截图、配置文件、日志记录等
- **执行记录**: 审批记录、培训记录、演练记录等
- **其他材料**: 合同协议、评估报告、测评证书等

## 🔧 整改建议

### 常见问题及整改方向

#### 1. 数据分类分级
- **问题**: 未建立分类分级制度
- **整改**: 制定《数据分类分级管理制度》，识别重要数据

#### 2. 数据加密
- **问题**: 敏感数据未加密存储/传输
- **整改**: 部署加密系统，启用 HTTPS/TLS

#### 3. 访问控制
- **问题**: 权限管理粗放，未遵循最小化原则
- **整改**: 实施 RBAC 权限模型，定期审查权限

#### 4. 安全审计
- **问题**: 未记录数据操作日志
- **整改**: 部署审计系统，记录关键操作

#### 5. 备份恢复
- **问题**: 备份策略不完善，未测试恢复
- **整改**: 制定备份策略，定期演练恢复
